# Contributing to PipelineIQ

Thank you for your interest in contributing to PipelineIQ! We welcome all types of contributions, from bug reports and documentation updates to new features and performance optimizations.

## 🛠️ Getting Started

1.  **Fork the Repository**: Create your own copy of the repository.
2.  **Clone Locally**: `git clone https://github.com/your-username/PipelineIQ.git`
3.  **Setup Environment**: 
    ```bash
    python -m venv venv
    source venv/bin/activate
    pip install -e .[dev]
    ```
4.  **Run Tests**: Ensure everything is working as expected by running `pytest`.

## 📝 Reporting Bugs

- Use the GitHub Issue Tracker to report bugs.
- Include a clear, descriptive title and as much relevant information as possible (logs, screenshots, steps to reproduce).

## 🚀 Proposing Features

- For major changes, please open an issue first to discuss what you'd like to change.
- Follow the MCP standard when adding new tools or resources.

## 💎 Pull Request Process

1.  Ensure all public functions have docstrings (Google Style).
2.  Add tests for any new functionality.
3.  Update the `CHANGELOG.md` with your changes under the `[Unreleased]` section.
4.  Maintain a clean commit history using Atomic Commits.
